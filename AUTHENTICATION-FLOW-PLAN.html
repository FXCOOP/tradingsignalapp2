Database Schema Changes Needed

Users Table Structure:

CREATE TABLE users ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), email VARCHAR(255) UNIQUE NOT NULL, password_hash VARCHAR(255) NOT NULL, full_name VARCHAR(255), phone VARCHAR(50), -- Access Control Fields access_level VARCHAR(20) DEFAULT 'free', -- 'anonymous', 'free', 'premium' free_signals_count INT DEFAULT 0, -- How many free signals they've viewed free_articles_count INT DEFAULT 0, -- How many free articles they've read -- Broker Integration Fields has_broker_account BOOLEAN DEFAULT false, broker_verified_at TIMESTAMP, broker_trader_id VARCHAR(255), -- Exness trader ID broker_name VARCHAR(100) DEFAULT 'exness', -- Tracking created_at TIMESTAMP DEFAULT NOW(), last_login_at TIMESTAMP, email_verified BOOLEAN DEFAULT false, -- Indexes for fast queries INDEX idx_email (email), INDEX idx_broker_trader_id (broker_trader_id) );

Activity Log Table (Optional but Recommended):

CREATE TABLE user_activity ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), user_id UUID REFERENCES users(id), action_type VARCHAR(50), -- 'view_signal', 'read_article', 'access_course' resource_type VARCHAR(50), -- 'signal', 'article', 'course' resource_id VARCHAR(255), created_at TIMESTAMP DEFAULT NOW(), INDEX idx_user_id (user_id), INDEX idx_created_at (created_at) );

Why this table? Track exactly what users view so you can enforce limits accurately and provide analytics.

Broker Postbacks Log Table (Security & Debugging):

CREATE TABLE broker_postbacks ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), user_id UUID REFERENCES users(id), broker_name VARCHAR(100), event_type VARCHAR(50), -- 'registration', 'deposit', 'trade' raw_data JSONB, -- Store complete postback payload processed BOOLEAN DEFAULT false, created_at TIMESTAMP DEFAULT NOW(), INDEX idx_user_id (user_id), INDEX idx_processed (processed) );

Security Best Practices
üîí CRITICAL SECURITY MEASURES:
Security Risk Protection Method Implementation
Fake postbacks Verify signature Check Exness signature header, validate IP whitelist
SQL injection Parameterized queries Use Supabase client (automatic protection)
Password leaks Bcrypt hashing Already using bcrypt with salt rounds = 12 ‚úì
Token theft HTTPS only, short expiry Set JWT expiry to 7 days, refresh on activity

User enumeration Generic error messages Don't say "email not found", say "invalid credentials"
