6
Test Your Postbacks

Exness provides a built-in testing tool!

  1. Go back to your postbacks list
  2. Click the edit icon next to each postback
  3. Click "Test all" button
  4. Scroll down to see the test results

‚úÖ What Success Looks Like:

You should see a 200 OK response with JSON data:

{ "success": true, "message": "Test postback received", "event": "registration", "timestamp": "2025-01-22T10:30:00Z" }

‚ùå Common Test Failures:

  • 404 Not Found: Check your URL is correct
  • 500 Server Error: Your API route has a bug
  • Timeout: Your server is too slow or offline
7
Verify in Your Database

After testing, check your Supabase database:

  1. Go to your Supabase project
  2. Click Table Editor
  3. Find exness_conversions table
  4. Look for test entries with event_type = "registration", "deposit", etc.

Or check your Admin Dashboard:

https://tradeflow.blog/admin/dashboard

Click the "Conversions" tab to see all postbacks

üîß Troubleshooting

‚ùå Problem: Postback Test Fails

Solution:

  1. Check your site is deployed and online: https://tradeflow.blog
  2. Test API route manually: https://tradeflow.blog/api/postback/exness
  3. Check Render logs for errors: https://dashboard.render.com
  4. Verify Supabase connection is working

‚ùå Problem: No Real Conversions Showing Up

Possible reasons:

  • Not enough time passed: Postbacks fire every 10 min to 2 hours
  • User didn't click your link: They must use YOUR affiliate link
  • Existing account: User already has Exness account (won't count)
  • Different email: User registered with different email than tracked

‚úÖ How to Verify Everything is Working:

  1. Test in Exness PA: Use built-in "Test all" button
  2. Check Supabase: Look for test entries in exness_conversions
  3. Real test: Click your link, register on Exness with NEW email, wait 10 minutes
  4. Check Admin Dashboard: Monitor conversions in real-time
